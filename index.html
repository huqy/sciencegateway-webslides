What is it?

Science Gateway refers to a web or mobile-based application enabling access to advanced technology cyberinfrastructures.

User-friendly interface
connecting from people to resources, and from people to people

Why Science Gateway?

  • Lower the use threshold for people

    User-friendly interface, packing the computer technology service into black box

  • An opportunity to share and collaborate

Science Gateway Examples

eBird

a science gateway example

A web portal for birdwatchers of all levels to record their avian sightings and upload them for future use by scientists.

  • Website: www.ebird.org
  • Host: The Cornell lab of ornithology
  • Impact: observ. from < 200K (mid 2005) to >4M (2013)

eBird Inside

Connects bird watchers to bird scientists

  • An interactive database as a core with a human/computer learning network
  • Machine learning to improve incoming data quality
  • Artificial Intelligence also used for data analyses
  • An active learning feedback loop formed to improve the entire system.

nanoHub

another science gateway example

A computational nanotechnology website for nano scientist to run simulations for their research and educational activities.

  • Website: nanohub.org
  • Host: Purdue University
  • Impact: 1.5M users, 500+ sim tools, 5500+ resources

Nanohub Inside

Hubzero with Rappture toolkit is the engine

  • Web server is written by PHP
  • Users access interactive, graphical tools via an applet in their Web browser.
  • Jobs can be dispatched to the national Grid infrastructure resources.

COSMIC2

a "young" science gateway example

A web-based platform for structural biologists to determine macromolecular structures using cryo-electron microscopy (cryo-EM)

  • Website: cosmic-cryoem.org
  • Host: University of Michigan, SDSC
  • Highlights: Handle 10-30 terabytes of data transfer

COSMIC2 Inside

Globus transfer with CIPRES Workbench

  • A user first uploads data and, can either use Globus Transfer or the browser.
  • The user then creates an analysis task and submit the task to HPC.
  • When the job is done, the user can view and download the output results.
  • The user can perform simple management steps on their data, tasks, and jobs.

Job Management

Airavata vs. Agave

Commons

  • Open source

    support both cloud and house-owned hosts

  • Blackbox

    to gateway, similar internal function modules

  • APIs

    multiple language SDKs

  • Funding

    good NSF support for years

  • Not good at

    no support for the individual resource configurations, yet.

And the winner goes to ... Agave

as our choice

Airavata Agave
API Interface Thrift-based RESTful
Adding resources Have to do it through service hosts Can add by users
Job status monitor Only through notification email Handled by Agave service itself
Job failure tolerance Experience submission failures occasionally in Comet Auto resubmit for a couple of times
Job deletion Can only delete the created-but-not-submitted jobs Can delete any queued jobs
File transport protocol Only SSH/STP proves working well so far Support multiple protocols
Job scheduler support Have critical problem in modifying the queue and parallel env since the template is hosted for all of the resources Only need a specific queue, other aspects can be circumvented
Tech support Less responsive, many features are claimed but not available Always immediate respond through Slack, much fewer to-be-done answers

Brainstorming

Ideas on how IDRE can have better supports on Science Gateway at UCLA

A centralized gateway?

A UCLA Agave tenant can be established and shared by multiple projects. IDRE will be gateway developers. Input parts have to be customizable by data readers and writers.

* * *

Marry Agave with Jupyter Notebook?

We can combine Jupyter's data exploration ability and Agave's job management. Also the binder-type Jupyter hub can be a good solution for post-processing.

IDRE gateway as a template?

Indivual projects will develop their own gateway based on the IDRE gateway. IDRE will collaborate and leverage the technical expertise and experience to research groups.

* * *

Blue ocean in Citizen Science?

Web application always has bigger advantage for connecting people to people. More opportunities and bigger markets might await to be explored.
